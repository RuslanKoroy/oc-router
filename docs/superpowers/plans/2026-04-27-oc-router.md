# OpenCode Router Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an npm package that provides an OpenCode plugin and companion CLI for routing each request across fast, balanced, and large model tiers.

**Architecture:** The plugin hooks `chat.message`, chooses a tier through forced prefixes or a router model, mutates the OpenCode user message model, and injects a visible routing banner. The CLI manages config files, installation into OpenCode config, model discovery, and diagnostics.

**Tech Stack:** TypeScript, Bun-compatible ESM, `@opencode-ai/plugin`, Node `fs/path/os/child_process`, `tsx`, `vitest`.

---

## File Structure

- `package.json`: package metadata, exports, binary, scripts, dependencies.
- `tsconfig.json`: strict ESM TypeScript configuration.
- `vitest.config.ts`: test configuration.
- `src/types.ts`: shared config, tier, model, and plugin context types.
- `src/model-id.ts`: parse and format OpenCode `provider/model` IDs.
- `src/config.ts`: load, merge, validate, and write router configs.
- `src/prompts.ts`: router prompt and JSON schema.
- `src/router.ts`: routing decision logic and fallback handling.
- `src/display.ts`: visible synthetic message and toast helpers.
- `src/plugin.ts`: OpenCode plugin entrypoint.
- `src/install.ts`: OpenCode config and command installer.
- `src/cli.ts`: `oc-router` command implementation.
- `src/index.ts`: public exports and package plugin shape.
- `tests/*.test.ts`: unit tests for config, routing, display, plugin, installer, CLI.
- `README.md`: installation, config, usage, and limitations.
